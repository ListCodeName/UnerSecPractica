📌 2.1

¿Qué es un Prompt?

Un prompt es la instrucción o mensaje que le enviamos a una Inteligencia Artificial para que genere una respuesta. Es nuestra forma de "hablar" con la máquina.

01 Definición de Prompt +

Un prompt (del inglés, "indicación" o "instrucción") es el texto que ingresamos en una herramienta de IA generativa para pedirle que realice una tarea. Puede ser una pregunta, una orden, una descripción o una combinación de todo eso.

El prompt es el punto de partida de toda interacción con una IA. La calidad de lo que obtenemos depende directamente de la calidad de lo que pedimos.

  • Entrada: el texto que escribimos (el prompt).
  • Procesamiento: la IA interpreta, descompone y analiza nuestra instrucción.
  • Salida: la respuesta generada por la IA basándose en el prompt.
02 ¿Por qué "Ingeniería" de Prompts? +

Se lo llama ingeniería porque no se trata de simplemente "preguntar algo", sino de diseñar, estructurar y optimizar las instrucciones de forma metódica para obtener el mejor resultado posible.

Al igual que un ingeniero diseña un puente considerando materiales, fuerzas y contexto, un ingeniero de prompts diseña sus instrucciones considerando:

  • Claridad: que la IA entienda exactamente qué se pide.
  • Contexto: proveer la información de fondo necesaria.
  • Especificidad: delimitar el alcance de la respuesta.
  • Formato: indicar cómo queremos la respuesta (lista, tabla, código, etc.).
  • Iteración: refinar el prompt basándose en las respuestas obtenidas.
03 El Contexto: la clave para las IAs +

Las IAs generativas no piensan ni tienen memoria persistente entre conversaciones. Cada interacción comienza desde cero. Por eso, el contexto que proporcionamos en el prompt es fundamental.

El contexto le dice a la IA:

  • Quién sos: "Soy un estudiante de secundaria de 4to año..."
  • Qué necesitás: "Necesito entender el concepto de..."
  • Para qué: "Para un trabajo práctico de la materia..."
  • Restricciones: "La respuesta debe tener máximo 200 palabras..."
  • Rol de la IA: "Actuá como un profesor de programación..."

Sin contexto, la IA "adivina" lo que querés y los resultados son genéricos. Con buen contexto, las respuestas son precisas, relevantes y útiles.

04 Los 5 Elementos de un buen Prompt +

Todo prompt efectivo combina estos elementos (no todos son obligatorios, pero cuantos más uses, mejor será el resultado):

  1. Rol: Asignar una identidad a la IA → "Actuá como un experto en..."
  2. Tarea: Describir claramente qué querés → "Explicame...", "Creá un...", "Compará..."
  3. Contexto: Dar información de fondo → "Para una clase de 4to año de secundaria..."
  4. Formato: Especificar cómo querés la respuesta → "En formato de lista con ejemplos..."
  5. Restricciones: Definir límites → "Máximo 300 palabras, en español neutro..."
🎯 2.2

Anatomía de un Prompt: de Malo a Excelente

Veamos el mismo tema — "crear una página web para un emprendimiento de pastelería" — abordado con tres niveles de calidad de prompt.

❌ Prompt Malo

Sin dirección, sin contexto

haceme una pagina web de pasteleria

¿Qué pasa? La IA no sabe qué tipo de página, qué estilo visual, qué contenido incluir, ni para qué plataforma. Generará algo genérico, probablemente incompleto, que no se ajustará a ninguna necesidad real.

✗ Sin rol asignado ✗ Sin contexto ✗ Sin formato esperado ✗ Sin restricciones ✗ Tarea vaga
⚡ Prompt Bueno

Claro pero mejorable

Creá el código HTML y CSS de una página web para una pastelería llamada "Dulce María". Debe tener un header con el logo, una sección de productos con fotos y precios, y un formulario de contacto. Usá colores pastel rosados.

Mejor, pero... Incluye la tarea clara y algo de formato, pero no define el público objetivo, no asigna un rol a la IA, no indica nivel de complejidad técnica ni restricciones de extensión.

✗ Sin rol ✓ Tarea clara ✓ Formato parcial ✗ Sin contexto de nivel ✗ Sin restricciones técnicas
🏆 Prompt Excelente

Completo, estructurado, eficiente

Actuá como un desarrollador web frontend senior.

Necesito que crees una landing page completa en HTML y CSS para una pastelería artesanal llamada "Dulce María", ubicada en Paraná, Entre Ríos.

Contexto: Es un emprendimiento familiar que vende tortas personalizadas, cupcakes y mesa dulce para eventos. El público objetivo son mujeres de 25 a 45 años.

Secciones requeridas:
1. Hero con imagen de fondo y slogan
2. Catálogo de productos (6 items con nombre, descripción corta y precio)
3. Sección "Sobre Nosotros" con historia del emprendimiento
4. Formulario de pedidos (nombre, teléfono, tipo de producto, fecha del evento)
5. Footer con redes sociales y WhatsApp

Restricciones:
- Diseño responsive (mobile-first)
- Paleta: rosa pastel (#f8b4c8), blanco y dorado
- Tipografía: Google Fonts (Playfair Display + Lato)
- Solo HTML + CSS, sin frameworks
- Código limpio y comentado

Resultado óptimo. La IA tiene toda la información necesaria para generar exactamente lo que necesitás, a la primera. Ahorra tiempo, iteraciones y frustración.

✓ Rol asignado ✓ Tarea específica ✓ Contexto completo ✓ Formato detallado ✓ Restricciones claras ★ Público objetivo ★ Datos reales
💻 2.3

Prompts para Programación

Cómo aprovechar la IA como asistente de desarrollo: conceptos básicos y recomendaciones prácticas.

🧱

Divide y Vencerás

No pidas "haceme una app completa". Dividí el problema en partes: primero la estructura HTML, luego los estilos, después la lógica. La IA trabaja mejor con tareas acotadas.

Estrategia
📋

Especificá el Lenguaje

Siempre indicá el lenguaje y las tecnologías: "en JavaScript vanilla", "usando Python 3.11", "con HTML semántico y CSS Grid". La IA adapta las soluciones al stack indicado.

Precisión
🔍

Pedí Explicaciones

Agregá "explicame línea por línea" o "agregá comentarios al código". No solo obtenés la solución: aprendés por qué funciona. Ideal para estudiantes.

Aprendizaje
🐛

Debugging con IA

Pegá tu código con el error y decí: "Este código me da [error exacto]. Explicame por qué falla y corregilo manteniendo la misma lógica." Contexto + error = solución rápida.

Debug
🔄

Iterá sobre la Respuesta

Si la primera respuesta no es perfecta, refiná: "Ahora modificá la función para que también acepte números negativos" o "Cambiá el diseño a modo oscuro." La conversación es iterativa.

Iteración
⚠️

Verificá Siempre

La IA puede generar código que "parece" correcto pero tiene errores lógicos. Siempre probá el código, entendé qué hace, y no copies sin comprender. La IA es tu asistente, no tu reemplazo.

Importante

📝 Ejemplo: Prompt de Programación Efectivo

💻 Prompt para Código
Actuá como profesor de programación web para alumnos de secundaria.

Necesito una función en JavaScript vanilla que:
1. Reciba un array de números
2. Filtre solo los números pares
3. Los ordene de menor a mayor
4. Devuelva el resultado

Requisitos:
- Usá arrow functions
- Agregá comentarios explicando cada paso
- Mostrá un ejemplo de uso con console.log
- Explicá brevemente qué hace cada método usado (filter, sort)

Este prompt produce código limpio, educativo y reutilizable. El alumno no solo obtiene la solución, sino que aprende los conceptos detrás de cada línea.

🎭 Asigná un Rol Técnico

"Actuá como desarrollador senior en React" produce código más profesional que un prompt genérico.

📐 Definí la Arquitectura

"Usá el patrón MVC", "Separar lógica de presentación", "Implementá con módulos ES6".

🧪 Pedí Tests

"Agregá 3 casos de prueba que validen la función" asegura que el código sea robusto.

📖 Pedí Alternativas

"Mostrá 2 formas diferentes de resolver esto" amplía tu conocimiento de los enfoques posibles.

🧩 Actividad Integradora

Desafío: Resolvé un Problema Real con Prompts

En esta actividad vas a abordar una problemática real desde múltiples ángulos, refinando tus prompts progresivamente.

🎯 Caso: Sistema de Gestión para una Biblioteca Escolar
Escenario: Tu escuela necesita un sistema digital para gestionar el préstamo de libros de la biblioteca. Actualmente todo se registra en papel y los libros se pierden, no se sabe quién los tiene, ni cuáles están disponibles. Tu tarea es diseñar la solución utilizando IA como asistente, dividiendo el problema en sub-problemas y atacando cada uno con prompts específicos.

📊 Ángulo 1: Análisis del Problema

Usá la IA para identificar todas las problemáticas del sistema actual y definir los requerimientos del nuevo sistema.

relevamiento requerimientos usuarios problemas actuales

🗃️ Ángulo 2: Diseño de Datos

Pedile a la IA que diseñe la estructura de datos: ¿qué información guardar de cada libro, alumno, préstamo?

base de datos entidades relaciones campos

🎨 Ángulo 3: Interfaz de Usuario

Diseñá con IA las pantallas principales: búsqueda de libros, registro de préstamo, panel del bibliotecario.

wireframe HTML/CSS usabilidad responsive

⚙️ Ángulo 4: Lógica y Reglas

Definí con IA las reglas del negocio: ¿cuántos libros puede llevar un alumno? ¿Cuántos días tiene? ¿Qué pasa si no devuelve?

reglas de negocio validaciones flujo de trabajo notificaciones

📌 Instrucciones de Entrega

  1. Elegí al menos 3 de los 4 ángulos para desarrollar.
  2. Para cada ángulo, escribí al menos 2 prompts progresivos: uno inicial y otro refinado basado en la respuesta obtenida.
  3. Documentá tanto tus prompts como las respuestas de la IA. Marcá qué elementos del prompt (rol, contexto, formato, restricciones) usaste en cada uno.
  4. Identificá al menos 2 errores o respuestas insuficientes que la IA te haya dado, y explicá cómo refinaste el prompt para mejorar el resultado.
  5. Escribí una reflexión final (mínimo 10 líneas) sobre qué aprendiste del proceso de refinamiento de prompts y cómo cambió tu forma de interactuar con la IA.
  6. Formato de entrega: documento digital (Google Docs, PDF o HTML) con capturas de pantalla de las conversaciones.
