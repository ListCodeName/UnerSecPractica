¿Qué es PHP?

// Hypertext Preprocessor — lenguaje de servidor

📌 Definición PHP (PHP: Hypertext Preprocessor) es un lenguaje de programación de propósito general orientado al desarrollo web, diseñado para ejecutarse del lado del servidor. PHP procesa el código antes de enviar la respuesta al navegador: el cliente nunca ve el código PHP, solo el HTML resultante.
Historia rápida
1994 — Rasmus Lerdorf crea las primeras herramientas PHP para rastrear visitas a su CV online.
1997 — PHP 3: reescritura completa. Adopción masiva como lenguaje web.
2004 — PHP 5: POO robusta, PDO, excepciones.
2015 — PHP 7: velocidad 2× respecto a PHP 5, tipado escalar.
2020–hoy — PHP 8.x: JIT compiler, match expressions, named arguments, fibers.
¿Dónde se ejecuta?
🖥 Lado del SERVIDOR
  • El código PHP corre en la máquina donde está instalado Apache/Nginx + PHP.
  • El navegador del cliente solo recibe el HTML final.
  • Acceso directo a bases de datos, sistema de archivos, variables de entorno.
  • El cliente no puede ver el código fuente PHP.
🚫 No funciona sin servidor
  • Un archivo .php abierto directamente en el navegador no se procesa.
  • Se necesita un servidor web con el módulo PHP activo (Apache, Nginx, etc.).
  • En desarrollo local: XAMPP, Laragon, WAMP o PHP built-in server (php -S).
Ventajas de PHP
  • ✅ Integración nativa con bases de datos (MySQL, PostgreSQL, SQLite…)
  • ✅ Sintaxis familiar para quienes conocen C, Java o JS
  • ✅ Soporte masivo en hostings compartidos (casi siempre incluido)
  • ✅ Gran ecosistema: Laravel, Symfony, WordPress, Drupal
  • ✅ Manejo de archivos, sesiones y cookies integrado
  • ✅ Código fuente oculto al cliente
  • ✅ PHP 8 con JIT es muy rápido para web
  • ✅ Curva de aprendizaje baja para iniciarse
Desventajas de PHP
  • ❌ Requiere servidor — no se puede usar directamente en el navegador sin infraestructura.
  • ❌ Inconsistencias históricas en nombres de funciones (str_len vs strlen) en versiones antiguas.
  • ❌ Tipado débil por defecto — puede generar bugs sutiles si no se usa declare(strict_types=1).
  • ❌ No es ideal para aplicaciones de tiempo real (WebSockets, chat) sin extensiones adicionales.
  • ❌ En proyectos mal mantenidos, el código puede volverse difícil de escalar ("código espagueti").

¿Por qué se necesita Apache?

// El flujo de una petición PHP

⚠ Diferencia fundamental Un archivo .html puede abrirse directamente en el navegador — es texto estático que el navegador interpreta. Un archivo .php contiene instrucciones que debe ejecutar un motor PHP en el servidor. Sin ese procesamiento previo, el navegador mostraría el código PHP en crudo o nada.

Con XAMPP en desarrollo local
// Pasos para ver un archivo PHP en el navegador con XAMPP 1. Abrir XAMPP Control Panel 2. Iniciar el módulo Apache (y MySQL si se necesita BD) 3. Guardar el archivo en: C:\xampp\htdocs\mi-proyecto\archivo.php 4. Abrir en el navegador: http://localhost/mi-proyecto/archivo.php // Alternativa: PHP Built-in Server (sin XAMPP, desde terminal) php -S localhost:8000 // Luego acceder a http://localhost:8000/archivo.php
💡 Diferencia con archivos HTML Los archivos .html estáticos se pueden abrir con doble clic desde el explorador de archivos porque el navegador los interpreta directamente. Los .php no: abrir un .php sin servidor mostrará texto plano o el código en crudo, según el navegador.

PHP vs JavaScript

// Comparativa: ejecución, seguridad y rendimiento

[PHP] — Servidor

🖥Corre en el servidor. El cliente nunca ve el código.
🔒Lógica de negocio y credenciales protegidas.
🗃Acceso directo a BD, archivos, email.
📄Genera el HTML antes de enviarlo.
🔃Cada petición puede recargar la página (sin JS).
⚙Requiere Apache / Nginx + motor PHP instalado.

[JavaScript] — Cliente (+ Node en servidor)

🌐Corre en el navegador. El código es visible con F12.
⚡Interactividad inmediata sin recargar la página.
🔓No se deben poner credenciales en JS del cliente.
📡Acceso a BD solo vía API (fetch/AJAX).
⏱Asincrónico nativo (Promises, async/await).
📦Con Node.js puede correr también en el servidor.

Aspecto PHP JavaScript (cliente)
¿Dónde corre? Servidor Navegador
¿El cliente ve el código? ❌ No (solo recibe HTML) ✅ Sí (F12 → Sources)
Acceso a base de datos ✅ Directo (PDO, MySQLi) ⚠ Solo via API/backend
Seguridad de credenciales ✅ Alta (código en servidor) ❌ Nunca poner passwords en JS cliente
Interactividad sin recarga ❌ Requiere JS o recarga ✅ Nativa (DOM, eventos)

💡 ¿Se usan juntos? Sí, y es lo más común. PHP genera el HTML del lado del servidor y JavaScript lo enriquece en el navegador. Por ejemplo: PHP trae los datos de la BD y los embebe en el HTML; JavaScript luego agrega filtros o animaciones sin recargar. Esta combinación es la base de gran parte de la web actual (WordPress, por ejemplo, usa PHP + JS).

Sintaxis PHP

// etiquetas de apertura, variables y asignación

Etiquetas PHP
El código PHP se escribe dentro de etiquetas <?php ... ?>. Todo lo que esté fuera de esas etiquetas se trata como HTML puro y se envía tal cual al navegador.
archivo.php<!-- HTML normal que el navegador recibe directamente --> <h1>Bienvenido al sitio</h1> <!-- Aquí comienza el código PHP --> <?php // PHP comienza con <?php — siempre obligatorio echo "<p>Este texto lo generó PHP</p>"; ?> <!-- Sigue el HTML normal --> <footer>Pie de página</footer>
Variables
En PHP todas las variables comienzan con el símbolo $. No se declara el tipo explícitamente (tipado dinámico): PHP infiere el tipo según el valor asignado. Los nombres son case-sensitive ($nombre ≠ $Nombre).
<?php // Declaración y asignación — siempre con $ delante del nombre $nombre = "Ana"; // string $edad = 28; // integer $precio = 1499.99; // float $activo = true; // boolean $nada = null; // null // Mostrar en pantalla con echo o print echo "Hola, " . $nombre . " — Tenés " . $edad . " años."; // Salida: Hola, Ana — Tenés 28 años. // El punto (.) es el operador de concatenación en PHP // También se puede usar interpolación dentro de comillas dobles: echo "Hola, {$nombre} — precio: {$precio}"; ?>
Constantes
<?php // Con define() — clásico, funciona en cualquier versión define("PI", 3.14159); define("NOMBRE_APP", "MiSistema"); // Con const — moderno, preferido en clases y namespaces const MAX_INTENTOS = 3; echo PI; // 3.14159 — las constantes NO llevan $ echo MAX_INTENTOS; // 3 ?>
Comentarios
<?php // Comentario de una línea (estilo C++ / JS) # Comentario de una línea (estilo Shell) /* Comentario de múltiples líneas */ ?>

Tipos de Dato

// Los 8 tipos primitivos de PHP con ejemplos

string

Cadena de texto

Secuencia de caracteres. Se delimita con comillas simples '...' o dobles "...". Las dobles permiten interpolación de variables.

int

Entero

Números sin parte decimal. Puede ser positivo, negativo o cero. Tamaño depende del SO (32 o 64 bits).

float

Flotante / Decimal

Números con parte decimal. También llamado double. Ej: 3.14, -0.5, 1.2e3.

bool

Booleano

Solo dos valores: true o false. Se usa para condiciones lógicas. Case-insensitive.

array

Arreglo

Colección de valores indexados (numérico) o asociativos (clave→valor). Muy flexibles en PHP.

null

Nulo

Representa la ausencia de valor. Una variable no asignada, asignada a null o eliminada con unset().

object

Objeto

Instancia de una clase. Agrupa propiedades y métodos. Base de la programación orientada a objetos en PHP.

callable

Callable / Closure

Funciones anónimas, lambdas o referencias a funciones que se pasan como parámetros.

Ejemplos de cada tipo
<?php // STRING $s1 = 'Hola mundo'; // comillas simples: sin interpolación $s2 = "Hola {$s1}"; // comillas dobles: con interpolación $lon = strlen($s1); // 10 — longitud del string // INTEGER $i1 = 42; $i2 = -100; $i3 = 0xFF; // 255 — hexadecimal $i4 = 0b1010; // 10 — binario // FLOAT $f1 = 3.1416; $f2 = 1.5e3; // 1500.0 — notación científica // BOOL $b1 = true; $b2 = false; var_dump($b1); // bool(true) // ARRAY indexado $frutas = ["manzana", "pera", "uva"]; echo $frutas[0]; // manzana // ARRAY asociativo (clave => valor) $persona = [ "nombre" => "Luis", "edad" => 32, "activo" => true ]; echo $persona["nombre"]; // Luis // NULL $vacio = null; var_dump(is_null($vacio)); // bool(true) // Verificar tipo en runtime gettype($i1); // "integer" gettype($f1); // "double" gettype($s1); // "string" ?>
Conversión (Casting)
<?php $txt = "42"; // string $num = (int) $txt; // 42 — cast a entero $dec = (float) "3.14"; // 3.14 $bool = (bool) 0; // false (0 es falsy) $str2 = (string) 99; // "99" // settype() modifica la variable directamente settype($txt, "integer"); // $txt ahora es 42 (int) ?>

Operadores aritméticos
Símbolo Nombre Ejemplo Resultado
+ Suma $a + $b / 10 + 5 15
- Resta $a - $b / 10 - 3 7

/ División $a / $b / 10 / 4 2.5
% Módulo (resto) $a % $b / 10 % 3 1

Operadores de asignación
Símbolo Equivale a Descripción
= $a = 5 Asignación simple
+= $a = $a + 3 Suma y asigna
-= $a = $a - 2 Resta y asigna
*= $a = $a * 4 Multiplica y asigna
/= $a = $a / 2 Divide y asigna
%= $a = $a % 3 Módulo y asigna
.= $s = $s . " más" Concatena y asigna (strings)
++ $a = $a + 1 Incremento en 1 (pre/post)
-- $a = $a - 1 Decremento en 1 (pre/post)
Operadores de comparación
⚠ == vs === == compara solo el valor (con conversión de tipo). === compara valor y tipo (estricto). Se recomienda usar === para evitar comparaciones inesperadas como 0 == "a" → true en PHP 7.
Símbolo Nombre Ejemplo Resultado
== Igual (valor) 5 == "5" true
=== Idéntico (valor + tipo) 5 === "5" false
!= Distinto (valor) 5 != 3 true
!== No idéntico 5 !== "5" true
< Menor que 3 < 5 true
> Mayor que 8 > 10 false
<= Menor o igual 5 <= 5 true
>= Mayor o igual 4 >= 7 false
<=> Spaceship (PHP 7+) 5 <=> 3 1 (izq > der)
Operadores lógicos
Símbolo Alternativa Nombre Descripción
&& and AND lógico Verdadero si ambas condiciones son verdaderas
|| or OR lógico Verdadero si al menos una es verdadera
! not NOT lógico Invierte el valor booleano
xor — XOR lógico Verdadero si exactamente una es verdadera
Operador de string · Concatenación
Símbolo Nombre Ejemplo Resultado

Ejemplo integrador
<?php $a = 10; $b = 3; // Aritméticos echo $a + $b; // 13 echo $a - $b; // 7 echo $a * $b; // 30 echo $a / $b; // 3.333... echo $a % $b; // 1 (resto de 10 / 3) echo $a ** $b; // 1000 (10 elevado a 3) // Asignación compuesta $a += 5; // $a = 15 $a *= 2; // $a = 30 $a++; // $a = 31 // Comparación $x = 5; $y = "5"; var_dump($x == $y); // bool(true) — mismos valores var_dump($x === $y); // bool(false) — tipos distintos // Lógicos $mayor = 18; $activo = true; if ($mayor >= 18 && $activo) { echo "Acceso permitido"; } // Concatenación de strings $msg = "Resultado: " . ($a + $b); $msg .= " unidades."; echo $msg; // Resultado: 33 unidades. ?>
