Elementos de una Base de Datos

Tablas, columnas y filas: los bloques fundamentales

📦
Base de Datos Contenedor lógico que agrupa tablas relacionadas. Ej: mi_empresa
📋
Tabla (Table) Estructura de filas y columnas. Cada tabla representa una entidad (Clientes, Productos).
📊
Columna (Column / Campo) Define un atributo y su tipo de dato: nombre VARCHAR(80)
➡️
Fila (Row / Registro) Un registro individual. Cada fila es una instancia de la entidad con todos sus valores.

DCL / TCL

Control de permisos y transacciones

DCL

Data Control Language

Controla quién puede hacer qué en la base de datos. Comandos: GRANT (otorgar permiso) y REVOKE (quitar permiso).

GRANT SELECT ON clientes TO usuario1;
— Le da permiso de solo lectura sobre la tabla clientes al usuario1.
REVOKE INSERT ON productos FROM usuario2;
— Le quita el permiso de insertar registros en productos al usuario2.
TCL

Transaction Control Language

Controla el flujo de transacciones: agrupa operaciones para que se ejecuten todas juntas o ninguna.

BEGIN / START TRANSACTION — Inicia una transacción.
COMMIT — Confirma todos los cambios realizados en la transacción.
ROLLBACK — Deshace todos los cambios si algo salió mal.
-- Transferencia bancaria segura
BEGIN;
  UPDATE cuentas SET saldo = saldo - 5000 WHERE id = 1;
  UPDATE cuentas SET saldo = saldo + 5000 WHERE id = 2;
COMMIT; -- Solo si ambas operaciones salieron OK
