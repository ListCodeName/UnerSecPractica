¿Qué es una SubConsulta?

Una consulta anidada dentro de otra consulta SQL

📌 Definición Una subconsulta (o subquery) es una instrucción SELECT que se encuentra dentro de otra instrucción SQL. La consulta interna se ejecuta primero y su resultado es utilizado por la consulta externa. También se las llama consultas anidadas.
🧠 ¿Por qué usarlas?
🎯

Filtrar con datos dinámicos

Cuando el valor del filtro no es un literal fijo sino que depende de otra consulta. Ej: "todos los produtos más caros que el promedio".

WHERE dinámico
📊

Reemplazar JOINs complejos

En algunos casos una subconsulta puede ser más legible que un JOIN, especialmente cuando solo necesitás verificar existencia.

Legibilidad
🔗

Consultas correlacionadas

Subconsultas que hacen referencia a la tabla de la consulta externa, ejecutándose una vez por cada fila procesada.

Correlacionada
📋

Usar como tabla virtual

Una subconsulta en el FROM actúa como una tabla temporal (tabla derivada) sobre la que se puede hacer SELECT.

Tabla derivada
📐 Sintaxis general
SELECT columna1, columna2 FROM tabla_principal WHERE columna OPERADOR ( SELECT columna FROM otra_tabla WHERE condicion );
⚠️ Reglas importantes
  • La subconsulta siempre va entre paréntesis ( ).
  • Si se usa con =, <, > etc., la subconsulta debe retornar exactamente un valor (escalar).
  • Si se usa con IN, la subconsulta puede retornar múltiples filas (una columna).
  • Si se usa con EXISTS, solo importa si la subconsulta retorna filas o no.
  • El ORDER BY no suele tener efecto dentro de una subconsulta (salvo casos específicos).
🔄 ¿Cuándo elegir subconsulta vs JOIN?
Situación Preferir Motivo
Comparar con un valor calculado (promedio, máximo) SubConsulta Más natural y legible
Combinar columnas de dos tablas en el resultado JOIN JOIN proyecta columnas de ambas tablas
Verificar si existe algún registro EXISTS Detiene la búsqueda al primer resultado
Obtener datos agrupados como tabla SubConsulta en FROM Actúa como tabla temporal (tabla derivada)
Una fila depende del contexto de la fila externa SubConsulta correlacionada Accede a valores de la fila actual
🗂️

Tipos de SubConsultas

Según dónde se ubican y cuántos valores retornan

📍 Por ubicación en la consulta
EN WHERE
La más común. Filtra filas según el resultado de la subconsulta. Puede usar =, IN, >, EXISTS, etc.
SELECT nombre FROM empleados WHERE salario > (SELECT AVG(salario) FROM empleados);
EN FROM
La subconsulta actúa como una tabla virtual. Se le debe asignar un alias. Muy útil para calcular subtotales y luego filtrarlos.
SELECT depto, total FROM (SELECT depto_id AS depto, COUNT(*) AS total FROM empleados GROUP BY depto_id) AS resumen WHERE total > 5;
EN SELECT
Devuelve un valor calculado para cada fila del resultado. La subconsulta debe retornar exactamente un valor (escalar).
SELECT nombre, (SELECT COUNT(*) FROM pedidos WHERE pedidos.cliente_id = c.id) AS total_pedidos FROM clientes c;
EN HAVING
Filtra grupos después del GROUP BY, usando el resultado de una subconsulta como umbral de comparación.
SELECT depto_id, AVG(salario) AS prom FROM empleados GROUP BY depto_id HAVING AVG(salario) > (SELECT AVG(salario) FROM empleados);
📏 Por cantidad de valores retornados
Tipo Retorna Operadores compatibles Ejemplo
Escalar 1 fila, 1 columna = < > <> (SELECT MAX(precio) FROM productos)
Columna Varias filas, 1 columna IN NOT IN ANY ALL (SELECT id FROM clientes WHERE pais = 'AR')
Tabla Varias filas, varias columnas EXISTS (en FROM con alias) (SELECT id, nombre FROM clientes WHERE activo = 1) AS c
Correlacionada Varía por fila externa EXISTS = IN Referencia a alias de la consulta externa
🔑 Operadores especiales con subconsultas
📋

IN / NOT IN

Comprueba si el valor pertenece (o no) al conjunto de resultados de la subconsulta. La más usada.

WHERE cliente_id IN (SELECT id FROM clientes WHERE pais = 'AR')
✅

EXISTS / NOT EXISTS

Retorna verdadero si la subconsulta devuelve al menos una fila. No importa qué columnas retorna, solo si hay resultado.

WHERE EXISTS (SELECT 1 FROM pedidos p WHERE p.cliente_id = c.id)
🔢

ANY / SOME

Verdadero si la comparación se cumple para al menos uno de los valores de la subconsulta. = ANY es igual a IN.

WHERE salario > ANY (SELECT salario FROM empleados WHERE depto_id = 3)
🔒

ALL

Verdadero si la comparación se cumple para todos los valores de la subconsulta. Equivale a mayor que el máximo (> ALL = > MAX).

WHERE salario > ALL (SELECT salario FROM empleados WHERE depto_id = 3)

Ejemplo Paso a Paso

Observá cómo el motor SQL resuelve una subconsulta internamente

🏪 Escenario: Tienda online Tenemos dos tablas: productos y pedidos. Queremos encontrar los productos cuyo precio es mayor al precio promedio de todos los productos.
📋 Tablas de ejemplo
Tabla: productos

2 Mouse Inalámbrico 4200 PC
3 Monitor 24" 32000 PC
4 Teclado Mecánico 11500 PC
5 Webcam HD 6800 Video
6 Hub USB-C 3900 Accesorios
Tabla: pedidos
id cliente producto_id cantidad
1 Ana Martínez 3 1
2 Luis García 1 2
3 María López 4 1
4 Carlos Díaz 2 3
5 Ana Martínez 6 1
🎬 Demostración interactiva

Usá los botones para avanzar y ver cómo el motor SQL resuelve la subconsulta paso a paso.

Paso 1 de 4
📝 Paso 1 — La consulta completa
SELECT nombre, precio FROM productos WHERE precio > (SELECT AVG(precio) FROM productos);
👀 Esta es la consulta que escribimos. La parte destacada en amarillo es la subconsulta. El motor SQL detecta que hay una subconsulta y la ejecutará primero antes de procesar la consulta principal.
⚙️ Paso 2 — Ejecución de la subconsulta
-- El motor ejecuta PRIMERO esta subconsulta: SELECT AVG(precio) FROM productos; -- Calcula el promedio de: 8500, 4200, 32000, 11500, 6800, 3900 -- Suma total = 66900 / 6 productos -- Resultado: 11150.00
🔢 El motor ejecuta la subconsulta de forma independiente. Calcula el promedio de todos los precios: (8500+4200+32000+11500+6800+3900) / 6 = 11150. Este valor escalar (un solo número) reemplazará a la subconsulta en la consulta externa.
🔄 Paso 3 — Sustitución del resultado
-- La subconsulta se reemplaza por su resultado: SELECT nombre, precio FROM productos WHERE precio > 11150; -- Ahora el motor evalúa cada fila: -- Auriculares BT : 8500 > 11150 → ❌ NO -- Mouse Inalámbrico: 4200 > 11150 → ❌ NO -- Monitor 24" : 32000 > 11150 → ✅ SÍ -- Teclado Mecánico : 11500 > 11150 → ✅ SÍ -- Webcam HD : 6800 > 11150 → ❌ NO -- Hub USB-C : 3900 > 11150 → ❌ NO
🔄 El resultado 11150 reemplaza a la subconsulta completa. La consulta externa filtra fila por fila: solo pasan las filas donde precio > 11150. El Monitor y el Teclado superan ese umbral.
✅ Paso 4 — Resultado final
-- Resultado de la consulta: SELECT nombre, precio FROM productos WHERE precio > (SELECT AVG(precio) FROM productos);
nombre precio
Monitor 24" 32000
Teclado Mecánico 11500
🎉 ¡Listo! Solo 2 productos tienen un precio superior al promedio (11150). El Motor ejecutó la subconsulta una sola vez, obtuvo el promedio y lo usó para filtrar las filas de la consulta principal. Sin subconsulta, habría que saber el promedio de antemano y "hardcodearlo" como literal.
➕ Ejemplo adicional: subconsulta con IN
🎯 Objetivo: Obtener los productos que sí fueron pedidos al menos una vez.
SELECT nombre, precio FROM productos WHERE id IN (SELECT DISTINCT producto_id FROM pedidos); -- La subconsulta retorna: (3, 1, 4, 2, 6) -- Se buscan los productos con id IN (1, 2, 3, 4, 6)
nombre precio
Auriculares BT 8500
Mouse Inalámbrico 4200
Monitor 24" 32000
Teclado Mecánico 11500
Hub USB-C 3900

📌 La Webcam HD (id=5) no aparece porque nunca fue pedida.

Ejercicios QUIZ

Escribí la consulta SQL en base al enunciado — debe usar subconsultas obligatoriamente

0 de 2 ejercicios resueltos
📋 Tablas disponibles para los ejercicios
empleados
id, nombre, salario, depto_id, fecha_ingreso
departamentos
id, nombre_depto, presupuesto
proyectos
id, nombre_proyecto, depto_id, costo
asignaciones
id, empleado_id, proyecto_id, horas
1
Empleados con salario mayor al promedio del departamento 2
📌 Enunciado completo Obtené el nombre y salario de todos los empleados cuyo salario supera el salario promedio de los empleados que pertenecen al departamento con id = 2. Ordená el resultado de mayor a menor salario.
-- Resultado esperado (ejemplo): nombre | salario ----------------|-------- Carlos Ruiz | 95000 María Fernández | 87500 Juan Pérez | 76000
✍️ Escribí tu consulta SQL (debe usar subconsulta):
💡 Pista: Usá una subconsulta escalar en el WHERE que calcule AVG(salario) filtrando por WHERE depto_id = 2. La consulta externa compara salario > (subconsulta).
✅ Una solución correcta: 
SELECT nombre, salario
FROM   empleados
WHERE  salario > (
           SELECT AVG(salario)
           FROM   empleados
           WHERE  depto_id = 2
       )
ORDER BY salario DESC;
2
Departamentos cuyo presupuesto supera el costo de todos sus proyectos
📌 Enunciado completo Listá el nombre del departamento y su presupuesto para aquellos departamentos cuyo presupuesto sea mayor a la suma total del costo de sus propios proyectos. Es decir, departamentos que "tienen fondos de sobra" después de cubrir todos sus proyectos.
-- Lógica de comparación: -- departamentos.presupuesto > SUM(proyectos.costo) del mismo depto -- Usá una subconsulta correlacionada con EXISTS o en WHERE
✍️ Escribí tu consulta SQL (debe usar subconsulta correlacionada):
💡 Pista: Usá una subconsulta correlacionada: en la subconsulta hacé referencia al alias de la consulta externa (d.id). La subconsulta calcula SUM(costo) de la tabla proyectos donde depto_id = d.id. Si un departamento no tiene proyectos, SUM retorna NULL → usá COALESCE.
✅ Una solución correcta: 
SELECT d.nombre_depto, d.presupuesto
FROM   departamentos d
WHERE  d.presupuesto > (
           SELECT COALESCE(SUM(p.costo), 0)
           FROM   proyectos p
           WHERE  p.depto_id = d.id
       );
