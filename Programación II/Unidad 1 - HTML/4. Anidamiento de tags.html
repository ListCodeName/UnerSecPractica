Unidad 1: Anidamiento de Tags

El anidamiento se refiere a poner unos elementos HTML dentro de otros. Es como una caja dentro de otra caja. Para que el navegador entienda nuestra página, debemos seguir reglas estrictas.

1. Estructura de Árbol (Padre e Hijo)

Cuando un tag está dentro de otro, se crea una relación:

  • Padre (Parent): El elemento contenedor.
  • Hijo (Child): El elemento contenido.
  • Hermanos (Siblings): Elementos que comparten el mismo padre.
<div>              <!-- Padre -->
    <p>Texto 1</p> <!-- Hijo del div, Hermano del siguiente p -->
    <p>Texto 2</p> <!-- Hijo del div -->
</div>

2. Regla de Oro: El orden de cierre

Los tags deben cerrarse en el orden inverso al que se abrieron (LIFO - Last In, First Out). "El último en abrir es el primero en cerrar".

Ejemplo Correcto ✔

<p>Texto en <strong>negrita y <em>cursiva</em></strong>.</p>

Aquí <em> está dentro de <strong>, y <strong> dentro de <p>.

Ejemplo Incorrecto ✘

<p>Texto en <strong>negrita y <em>cursiva</strong></em>.</p>

¡Error! Aquí intentamos cerrar <strong> antes de cerrar su hijo <em>. Esto puede "romper" el diseño.

3. Elementos de Bloque vs. En Línea

Generalmente:

  • Los elementos de bloque (div, p, ul, section) pueden contener otros elementos de bloque y elementos en línea.
  • Los elementos en línea (span, a, strong) solo pueden contener otros elementos en línea y texto.

Regla común: No pongas un elemento de bloque dentro de un elemento en línea.

<!-- MAL -->
<a href="#"> <div>Clickeame</div> </a> (La mayoría de las veces es mala práctica, aunque HTML5 es más flexible con <a>)

<!-- MAL -->
<p>Este es un párrafo con un <ul><li>lista</li></ul> dentro.</p> (Un p no puede contener una lista ul)

4. Indentación

Para leer mejor el código y ver el anidamiento, usamos la indentación (sangría). Cada nivel de anidamiento debe moverse hacia la derecha.

<html>
    <body>
        <div>
            <p>Hola</p>
        </div>
    </body>
</html>
