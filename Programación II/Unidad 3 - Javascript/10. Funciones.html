Unidad 3: Funciones en JavaScript

Las funciones son uno de los bloques de construcción fundamentales en JavaScript. Una función es un conjunto de instrucciones que realiza una tarea o calcula un valor. Para usar una función, primero debes definirla y luego llamarla (invocarla).

1. Declaración de Función (Function Declaration)

Es la forma más clásica. Se utiliza la palabra clave function seguida del nombre de la función. 

function saludar(nombre) {
    return "Hola, " + nombre + "!";
}

// Invocación
console.log(saludar("Ana")); // Salida: Hola, Ana!
Hoisting: Las funciones declaradas de esta forma se "elevan" al inicio, por lo que puedes llamarlas antes de definirlas en el código.

2. Expresión de Función (Function Expression)

Aquí, una función se asigna a una variable. A menudo son funciones anónimas (sin nombre).

const sumar = function(a, b) {
    return a + b;
};

console.log(sumar(5, 3)); // Salida: 8

3. Funciones Flecha (Arrow Functions)

Introducidas en ES6, son una sintaxis más corta y moderna. Son muy útiles para funciones simples.

// Forma clásica
const multiplicar = function(a, b) {
    return a * b;
};

// Forma flecha
const multiplicarFlecha = (a, b) => {
    return a * b;
};

// Si es una sola línea, el return es implícito
const dividir = (a, b) => a / b;

console.log(multiplicarFlecha(4, 5)); // Salida: 20
console.log(dividir(10, 2)); // Salida: 5

4. Parámetros y Argumentos

Los parámetros son las variables listadas en la definición de la función. Los argumentos son los valores reales que se pasan a la función cuando se invoca.

function presentar(nombre, edad = 18) { // edad tiene un valor por defecto
    console.log(`Soy ${nombre} y tengo ${edad} años.`);
}

presentar("Carlos", 25); // "Soy Carlos y tengo 25 años."
presentar("Lucía");      // "Soy Lucía y tengo 18 años."

5. Usos Concretos y Buenas Prácticas

A. Modularización y Reutilización (DRY)

El principio "Don't Repeat Yourself" (No te repitas). Si tienes código que se repite en varios lugares (por ejemplo, calcular el IVA), crea una función.

function calcularPrecioFinal(precio) {
    const iva = 0.21;
    return precio + (precio * iva);
}

// Usamos la misma lógica para diferentes productos
let precioCamisa = calcularPrecioFinal(2000);
let precioPantalon = calcularPrecioFinal(4500);

B. Funciones Puras (Transformación de Datos)

Funciones que, dada una misma entrada, siempre retornan la misma salida y no tienen efectos secundarios (no modifican variables externas). Son predecibles y fáciles de testear.

// Función Pura
function cuadrado(n) {
    return n * n;
}

// Función Impura (depende de algo externo)
let total = 0;
function agregarAlTotal(cantidad) {
    total += cantidad; // Modifica una variable externa
}

C. Encapsulamiento de Lógica

Ocultar la complejidad de una operación. El usuario de la función no necesita saber CÓMO se hace, solo qué necesita pasarle y qué recibe.

function obtenerNombreCompleto(usuario) {
    // Lógica interna para formatear o capitalizar
    let n = usuario.nombre.charAt(0).toUpperCase() + usuario.nombre.slice(1);
    let a = usuario.apellido.charAt(0).toUpperCase() + usuario.apellido.slice(1);
    return `${n} ${a}`;
}
