Introducción a la OOP

La Programación Orientada a Objetos es un paradigma que organiza el código en torno a objetos: entidades que combinan datos (atributos) y comportamiento (métodos).

En JavaScript, las clases (introducidas en ES6) permiten definir estos objetos de forma clara y estructurada.

Atributos Métodos Constructor Get / Set
Conceptos Clave

Atributos: Variables internas que almacenan el estado del objeto (nombre, edad, etc.).

Métodos: Funciones que definen el comportamiento del objeto.

Constructor: Método especial (constructor()) que se ejecuta al crear una instancia con new.

Get / Set: Accesores que permiten leer y modificar atributos de forma controlada.

Ejemplo: Clase Persona
class Persona { constructor(nombre, edad) { this._nombre = nombre; // atributo (convención privado) this._edad = edad; } // Getter get nombre() { return this._nombre; } // Setter con validación set edad(valor) { if (valor > 0) this._edad = valor; else console.log("Edad inválida"); } saludar() { return `Hola, soy ${this._nombre} y tengo ${this._edad} años.`; } }

Abstracción

La abstracción consiste en mostrar solo la información esencial y ocultar los detalles de implementación. Permite modelar objetos del mundo real enfocándonos en qué hacen, no en cómo lo hacen.

JavaScript no tiene clases abstractas nativas, pero podemos simularlas lanzando errores en métodos que deben ser implementados por subclases.

Simulación en JavaScript
class Figura { calcularArea() { throw new Error("Método abstracto: implementar en subclase"); } } class Circulo extends Figura { constructor(radio) { super(); this.radio = radio; } calcularArea() { return (Math.PI * this.radio ** 2).toFixed(2); } }
Comparación con Java

Java sí soporta clases abstractas de forma nativa con la keyword abstract:

JavaScript (simulado)
// No hay keyword abstract class Figura { calcularArea() { throw new Error("Abstracto"); } }
Java (nativo)
abstract class Figura { abstract double calcularArea(); } class Circulo extends Figura { double radio; double calcularArea() { return Math.PI * radio * radio; } }

Encapsulamiento

El encapsulamiento protege los datos internos de un objeto, exponiendo solo lo necesario a través de una interfaz pública. Esto previene modificaciones accidentales y asegura la integridad del estado.

Desde ES2022, JavaScript soporta campos privados reales usando el prefijo #.

Campos Privados en JS
class CuentaBancaria { #saldo; // campo privado real constructor(titular, saldoInicial) { this.titular = titular; this.#saldo = saldoInicial; } depositar(monto) { if (monto > 0) this.#saldo += monto; } getSaldo() { return this.#saldo; } }

Herencia

La herencia permite que una clase hija (subclase) adquiera los atributos y métodos de una clase padre (superclase). Esto promueve la reutilización de código y la jerarquía lógica.

En JavaScript se usa extends y super() para invocar al constructor padre.

Herencia en JS
class Animal { constructor(nombre) { this.nombre = nombre; } hablar() { return `${this.nombre} hace un sonido.`; } } class Perro extends Animal { hablar() { return `${this.nombre} ladra: ¡Guau!`; } } class Gato extends Animal { hablar() { return `${this.nombre} maulla: ¡Miau!`; } }

Polimorfismo

El polimorfismo permite que diferentes clases respondan al mismo método de formas distintas. En JS existe naturalmente gracias al tipado dinámico (duck typing).

Java soporta también sobrecarga de métodos (mismo nombre, distintos parámetros), algo que JavaScript no tiene de forma nativa.

Polimorfismo en JS
class Forma { describir() { return "Soy una forma genérica."; } } class Triangulo extends Forma { describir() { return "▲ Soy un Triángulo con 3 lados."; } } class Cuadrado extends Forma { describir() { return "■ Soy un Cuadrado con 4 lados iguales."; } } // Misma interfaz, distinto comportamiento: [new Triangulo(), new Cuadrado()].forEach( f => console.log(f.describir()) );
Sobrecarga en Java vs JS
JavaScript (sin sobrecarga)
// JS no distingue por parámetros class Calc { sumar(a, b, c) { return a + b + (c || 0); } }
Java (sobrecarga nativa)
class Calc { int sumar(int a, int b) { return a + b; } int sumar(int a, int b, int c) { return a + b + c; } }
