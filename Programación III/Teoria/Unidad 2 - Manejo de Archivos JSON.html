¿Qué es JSON?

JSON (JavaScript Object Notation) es un formato ligero de intercambio de datos. Aunque nació de JavaScript, hoy es el estándar universal para transmitir información entre servidores, APIs y aplicaciones.

Fue creado por Douglas Crockford a principios de los años 2000 como alternativa más simple a XML. Su sintaxis es fácil de leer para humanos y simple de interpretar para las máquinas.

Ligero Intercambio Universal Legible
Origen y contexto

Antes de JSON, el formato dominante era XML, mucho más verboso y pesado. Crockford propuso JSON como un subconjunto de la notación de objetos de JavaScript, logrando un formato más compacto.

En 2013, JSON fue estandarizado como ECMA-404 y luego como RFC 8259. Hoy lo usan el 99% de las APIs web modernas (REST APIs), archivos de configuración (package.json, tsconfig.json) y bases de datos como MongoDB.

Dato curioso — JSON vs XML

El mismo dato en XML ocupa aproximadamente un 30-40% más de espacio que en JSON. Por eso las APIs modernas migraron masivamente a JSON: menos bytes = más velocidad = menos costo de transferencia.

Sintaxis de JSON

JSON se compone de dos estructuras básicas: objetos (pares clave-valor entre llaves) y arrays (listas ordenadas entre corchetes).

Estructura de un archivo JSON
{ "nombre": "María López", "edad": 28, "activo": true, "cursos": ["JavaScript", "Python", "SQL"], "direccion": { "ciudad": "Paraná", "provincia": "Entre Ríos" }, "telefono": null }
Reglas de sintaxis

📌 Las claves siempre van entre comillas dobles ("clave").

📌 Los valores pueden ser: string, number, boolean (true/false), null, object o array.

📌 No se permiten: comentarios, comas finales (trailing commas), comillas simples ni funciones.

📌 JSON no es igual a un objeto JS: un objeto JS es más flexible (admite funciones, claves sin comillas, etc.).

JSON.parse() y JSON.stringify()

Estas son las dos funciones clave para trabajar con JSON en JavaScript:

// Convertir string JSON → Objeto JavaScript const jsonTexto = '{"nombre":"Ana","edad":22}'; const objeto = JSON.parse(jsonTexto); console.log(objeto.nombre); // "Ana" // Convertir Objeto JavaScript → string JSON const persona = { nombre: "Carlos", edad: 30 }; const jsonString = JSON.stringify(persona); console.log(jsonString); // '{"nombre":"Carlos","edad":30}' // Con formato legible (indentado) console.log(JSON.stringify(persona, null, 2));
Operaciones con JSON

Cuando trabajamos con JSON como una mini base de datos en el navegador, las operaciones principales son: leer, agregar, modificar y eliminar elementos.

Datos iniciales
// Simulamos una "base de datos" de alumnos en JSON let alumnos = [ { "id": 1, "nombre": "Ana García", "nota": 8.5 }, { "id": 2, "nombre": "Luis Pérez", "nota": 7.0 }, { "id": 3, "nombre": "María López", "nota": 9.2 } ];
Leer / Buscar datos
// Listar todos alumnos.forEach(a => console.log(`${a.nombre} - Nota: ${a.nota}`)); // Buscar por ID const buscado = alumnos.find(a => a.id === 2); console.log(buscado); // {id:2, nombre:"Luis Pérez", nota:7.0} // Filtrar aprobados (nota >= 7) const aprobados = alumnos.filter(a => a.nota >= 7);
Insertar un nuevo elemento
// Crear un nuevo alumno const nuevo = { id: 4, nombre: "Pedro Ruiz", nota: 6.8 }; // Agregarlo al array alumnos.push(nuevo); // Verificar console.log(JSON.stringify(alumnos, null, 2));
Modificar un elemento existente
// Actualizar la nota del alumno con ID 2 const alumno = alumnos.find(a => a.id === 2); if (alumno) { alumno.nota = 8.0; // actualizar directamente console.log("Nota actualizada:", alumno); }
Eliminar un elemento
// Eliminar alumno con ID 3 alumnos = alumnos.filter(a => a.id !== 3); // El array ahora no contiene a María López console.log(alumnos);
Persistir en localStorage

En el navegador, podemos usar localStorage para guardar datos JSON de forma persistente (sobrevive al cerrar el navegador):

// Guardar en localStorage localStorage.setItem("alumnos", JSON.stringify(alumnos)); // Recuperar de localStorage const datos = JSON.parse(localStorage.getItem("alumnos")); // Eliminar del localStorage localStorage.removeItem("alumnos");

💡 localStorage solo almacena strings, por eso necesitamos JSON.stringify() al guardar y JSON.parse() al leer.

Datos Curiosos sobre JSON

🌍 JSON domina las APIs del mundo

Más del 99% de las APIs REST modernas usan JSON como formato de intercambio. Servicios como Twitter/X, GitHub, Spotify, Google Maps y prácticamente toda aplicación web que usás envía y recibe JSON.

📦 package.json: el corazón de todo proyecto Node.js

Cada proyecto JavaScript/Node.js tiene un archivo package.json que define nombre, versión, dependencias y scripts. npm (con más de 2 millones de paquetes) depende enteramente de JSON para funcionar.

🔐 JSON Web Tokens (JWT) en seguridad

Los JWT son tokens de autenticación usados por millones de aplicaciones. Están compuestos por tres partes codificadas en Base64, y el payload (la parte central con los datos del usuario) es un objeto JSON. Es el estándar para login con OAuth, APIs protegidas y sesiones sin cookies.

🗄️ MongoDB: una base de datos JSON

MongoDB, una de las bases de datos más populares del mundo, almacena sus documentos en BSON (Binary JSON). Básicamente, toda tu base de datos es una colección gigante de objetos JSON. Empresas como Uber, eBay y Forbes la usan.

⚙️ Archivos de configuración en todas partes

VS Code (settings.json), TypeScript (tsconfig.json), ESLint (.eslintrc.json), Chrome extensions (manifest.json), GitHub Actions... Casi toda herramienta moderna usa JSON para su configuración.

📡 JSON + IoT

Los dispositivos IoT (sensores, cámaras, wearables) envían datos en formato JSON a través de protocolos como MQTT. Un sensor de temperatura podría enviar: {"sensor":"temp01","valor":23.5,"unidad":"°C"}. Así de simple es el lenguaje de las «cosas conectadas».

Ejercicio Práctico
Mini Base de Datos: Gestión de Productos

En este ejercicio vas a crear un sistema simple de gestión de productos usando JSON y JavaScript puro. El objetivo es practicar las operaciones CRUD (Crear, Leer, Actualizar, Eliminar) que aprendimos.

Consigna

Partiendo de los siguientes datos iniciales, implementá las funciones pedidas:

const productos = [ { "id": 1, "nombre": "Cuaderno A4", "precio": 1500, "stock": 25 }, { "id": 2, "nombre": "Lapicera azul", "precio": 350, "stock": 100 }, { "id": 3, "nombre": "Resaltador amarillo", "precio": 800, "stock": 40 }, { "id": 4, "nombre": "Mochila escolar", "precio": 15000, "stock": 8 }, { "id": 5, "nombre": "Calculadora científica", "precio": 12000, "stock": 12 } ];
Funciones a implementar

1. listarProductos() — Mostrar todos los productos en consola con formato: "[ID] Nombre - $Precio (Stock: X)"

2. buscarPorNombre(texto) — Recibir un texto y devolver todos los productos cuyo nombre contenga ese texto (búsqueda parcial, sin distinguir mayúsculas).

3. agregarProducto(nombre, precio, stock) — Agregar un nuevo producto con ID autoincremental (el siguiente al último ID existente).

4. actualizarPrecio(id, nuevoPrecio) — Encontrar un producto por ID y actualizar su precio. Si no existe, mostrar error.

5. eliminarProducto(id) — Eliminar un producto por ID. Retornar el producto eliminado o un mensaje de error.

6. productosMasCaros(limite) — Devolver los productos ordenados de mayor a menor precio, limitados a la cantidad indicada.

7. guardarEnLocalStorage() — Guardar el array de productos en localStorage.

8. cargarDesdeLocalStorage() — Recuperar los productos desde localStorage (si existen) y cargarlos en el array.

Bonus (opcional)

Implementá una función exportarJSON() que genere un archivo JSON descargable con los datos actuales. Tip: usá Blob y URL.createObjectURL().
